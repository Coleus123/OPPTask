(здесь картинка)
Пользователь: 0

Бот: Задача № 2 …

Бот: Задание № 27
Имеется набор данных, состоящий из пар положительных целых чисел. Необходимо выбрать из каждой пары ровно одно число так, чтобы сумма всех выбранных чисел не делилась на 3 и при этом была минимально возможной. Гарантируется, что искомую сумму получить можно. Программа должна напечатать одно число  — минимально возможную сумму, соответствующую условиям задачи.
Входные данные.
Файл A
Файл B
Даны два входных файла (файл A и файл B), каждый из которых содержит в первой строке количество пар N (1 ≤ N ≤ 100000). Каждая из следующих N строк содержит два натуральных числа, не превышающих 10 000.
Пример организации исходных данных во входном файле:

1 3
5 12

Предупреждение: для обработки файла B не следует использовать переборный алгоритм, вычисляющий сумму для всех возможных вариантов, поскольку написанная по такому алгоритму программа будет выполняться слишком долго.
4 задача 
(идея) Хранить статистику каждого пользователя о пройденных тестах(средний балл за несколько последних пройденных вариантов, среднее время выполнения и т.д.)
Формулировка:
Добавляется новая команда /statistic, она не принимает параметры.
Бот при запросе пользователя команды /statistic будет выдавать ему статистику за последние 5 решенных вариантов по каждому предмету: средний балл, среднее время.
Пример:
Пользователь: /statistic
Бот: Статистика за последние 5 вариантов по каждому предмету:
Математика: средний балл 40/80, среднее время выполнения задач - 20 минут 40 секунд.
Русский язык: средний балл 27/95, среднее время выполнения задач - 46 минут 52 секунд.
Информатика: не решено ни одного варианта.
Пользователь: 67088 200157478
Бот: Ответ правильный
Бот: Тест завершен за 1 час 50 минут 30 секунд. Правильное количество ответов - 22/27. Пройти тест заново или выйти?
Пользователь: /exitTest
